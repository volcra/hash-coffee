Usage

How to get started

Getting Started

You need first to get #Coffee, go to the Download page for more information on how to install and set up #Coffee.

Getting Help

Run #Coffee without any options or passing -h or --help to get help.

$>build\install\hash-coffee\bin\hash-coffee
usage: #coffee [options]
Options
 -b,--bare           Compile the JavaScript without the top-level function
                     safety wrapper.
 -c,--compile        Compile a .coffee script into a .js JavaScript file
                     of the same name.
 -e,--eval           Compile and print a little snippet of CoffeeScript
                     directly from the command line.
 -h,--help           Print Help.
 -j,--join <FILE>    Before compiling, concatenate all scripts together in
                     the order they were passed, and write them into the
                     specified file. Useful for building large projects.
 -o,--output <DIR>   Write out all compiled JavaScript files into the
                     specified directory.
 -p,--print          Instead of writing out the JavaScript as a file,
                     print it directly to stdout.

Command-Line Usage

Options

-b,--bare

Compile the JavaScript without the top-level function safety wrapper.

$>build\install\hash-coffee\bin\hash-coffee -b 'console.log num for num in [10..1]'
var num, _i;

for (num = _i = 10; _i >= 1; num = --_i) {
  console.log(num);
}
The safety wrapper is the anonymous function around your code. Compare to the output above. 
(function() {
  var num, _i;

  for (num = _i = 10; _i >= 1; num = --_i) {
    console.log(num);
  }

}).call(this);

-c,--compile

Compile a .coffee script into a .js JavaScript file of the same name.

-e,--eval

Compile and print a little snippet of CoffeeScript directly from the command line.

$>build\install\hash-coffee\bin\hash-coffee -e 'console.log num for num in [10..1]'
(function() {
  var num, _i;

  for (num = _i = 10; _i >= 1; num = --_i) {
    console.log(num);
  }

}).call(this);

-j,--join <FILE>

This is usefull for projects with multiple sources. Before compiling, the list of files will be joined togheter in the order they were passed.

-o,--output <DIR>

By default compiled JS are written to the same location fo the LESS style sheet. To change that behavior use this option.

-p,--print

Write the compiled JS to stdout.

You may send the output to a file of your choice.

$>build\install\hash-coffee\bin\hash-coffee -p src\test\resources\test.coffee > test.js

Sample File

This file is located at ${basedir}\src\test\resources\test.coffee. Where basedir is the path where you unpacked the application.

# testing coffeescript

class Greet
    constructor: (@who) ->

    salute: () ->
        console.log "Hello #{@who}"

g = new Greet "world"
g.salute()

Then execute the hash-coffee command. The option -p will instruct the command to print the result to the console.

This will compile the source file test.coffee and produce the following output.

$>build\install\hash-coffee\bin\hash-coffee -p -b src\test\resources\test.coffee
var Greet, g;

Greet = (function() {
  function Greet(who) {
    this.who = who;
  }

  Greet.prototype.salute = function() {
    return console.log("Hello " + this.who);
  };

  return Greet;

})();

g = new Greet("world");

g.salute();
